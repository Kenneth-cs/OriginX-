# 源极科技官网 - 全站简易后台设计方案 (综合完整版)

结合您对官网可配置化模块的全面梳理，我们对原有的“新闻后台”方案进行了升级，将其扩展为一个**覆盖全站核心内容的简易 CMS 系统**。

依然采用我们敲定的**“轻量、快、省钱”架构**：手写极简 Node.js (Express) 后端 + SQLite 数据库。

---

## 1. 系统核心功能模块规划

为了让运营人员能够随时脱离代码更新官网内容，后台系统将划分为以下四个核心模块：

### 模块一：资讯与内容发布管理 (News CMS)
- **功能**：管理 `News.tsx` 页面的动态文章列表和详情。
- **可配项**：文章标题、摘要、分类（如“行业洞察”）、发布日期、正文（Markdown 编辑，支持图文混排）、封面图、状态（发布/草稿/下线）。

### 模块二：首页配置管理 (Home CMS)
- **功能**：管理 `Home.tsx` 的各大结构化文本与板块。
- **Hero 首屏区**：顶部 Tag、核心大标题（支持拆分）、滚动关键词数组、主描述文案及底部金句、按钮文案及链接。
- **三大业务矩阵**：可动态增删矩阵卡片（目前为 3 个），配置图标、主标题、描述文案、子业务列表。
- **源极实验室**：模块介绍文案、标签云数组。
- **联络中心与终端**：地理位置标签、命令行服务列表数组（分创作者/企业两类）、接收表单的邮箱地址、商务微信展示（支持二维码图片）。

### 模块三：品牌与关于页面配置 (Brand CMS)
- **功能**：管理 `Brand.tsx` 的静态展示文案。
- **可配项**：品牌理念描述（“源”与“极”的阐释）、视觉解构说明（中心星核、交错轨道、隐晦泛光）、色彩美学文案（深空蓝与天际青）。

### 模块四：全局基础设置 (Global Settings)
- **功能**：对应 `Layout.tsx` 以及网站的全局配置。
- **SEO 与 TDK**：配置全局及各独立页面的 Title（网站标题）、Description（网站描述）、Keywords（关键词）。*(配合我们将要引入的 react-helmet-async 动态生效)*
- **页脚 (Footer)**：版权年份、备案号、隐私政策/服务条款链接。
- **导航栏 (Navbar)**：顶部导航菜单项的展示与隐藏。

## 2. 数据库表结构设计 (SQLite)

为了支持上述结构化和非结构化数据的灵活存储，我们设计以下两张核心表：

### 表 1：`news` (资讯文章表)
专门用于存储动态的新闻/文章列表。
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `id` | INTEGER | 主键，自增 |
| `title` | TEXT | 文章标题 |
| `summary` | TEXT | 文章摘要 |
| `content` | TEXT | 正文内容 (Markdown 格式) |
| `cover_image`| TEXT | 封面图 URL 路径 |
| `category` | TEXT | 分类 |
| `status` | TEXT | 状态 (draft / published) |
| `publish_date`| TEXT | 发布日期 |
| `created_at` | TEXT | 创建时间 |
| `updated_at` | TEXT | 最后修改时间 |

### 表 2：`settings` (全局配置与单页配置表)
采用 Key-Value (键值对) 的形式存储结构化 JSON，方便无限扩展而不需要频繁改数据库字段。
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `key` | TEXT | 配置的唯一标识 (如 `home_hero`, `global_seo`, `brand_concept`) |
| `value`| TEXT | 配置项的 JSON 字符串数据 |
| `updated_at`| TEXT | 最后修改时间 |

*(例如：`key: 'home_hero'` 对应的 `value` 可能是一个包含 title, tags, description 的 JSON 对象)*

## 3. 后端 API 接口规划 (Express)

*   `POST /api/auth/login`：管理员登录。
*   **News 模块**：
    *   `GET /api/news` / `GET /api/news/:id`：获取新闻。
    *   `POST /api/news` / `PUT /api/news/:id` / `DELETE /api/news/:id`：管理新闻。
*   **Settings 配置模块**：
    *   `GET /api/settings`：获取所有配置（供前端首页、品牌页等渲染使用）。
    *   `GET /api/settings/:key`：获取单个配置块。
    *   `PUT /api/settings/:key`：更新单个配置块。
*   **文件上传**：
    *   `POST /api/upload`：图片等资源上传。

## 4. 开发步骤更新与 TODO

为了将全站配置纳入开发，我们需要对前端的实现方式做较大改造：首页和品牌页不能再写死在代码里了。

1. **后端数据库与 API 开发**：建立 `news` 和 `settings` 两张表，提供读写接口。
2. **前端后台管理界面开发**：除了新闻编辑列表，还需要开发“页面配置”表单，供运营人员修改 JSON 配置（我们会做成可视化的表单界面）。
3. **前端展示页重构**：改造 `Home.tsx`、`Brand.tsx`、`Layout.tsx`，在组件加载时 (`useEffect` 或 React Router 的 `loader`) 从 `/api/settings` 获取数据进行渲染。
4. **接入 SEO (react-helmet-async)**：将获取到的 SEO 数据动态注入页面。
