# 源极科技官网 - 本次开发进度与逻辑说明文档

## 1. 整体开发进度概述
本次开发将原本纯静态的前端页面成功改造为**具备完整增删改查能力的动态全栈网站**，并完成了**生产环境（腾讯云）的部署上线**与**正式域名绑定**。

### 已完成的核心开发节点：
- [x] **后端环境搭建**：基于 Node.js + Express，搭建轻量级 API 接口。
- [x] **数据库设计**：采用本地轻量级 SQLite 数据库，建立 `news`（新闻内容）、`settings`（全局配置）、`contacts`（客户线索）三大核心表。
- [x] **后台管理系统**：开发了一套内部使用的管理后台，具备 JWT 鉴权功能。
- [x] **前端动态化改造**：将首页（部分文案通过接口读取）、源极视界（全量读取数据库）与联络中心（真实表单提交）接入后端 API。
- [x] **品牌视觉深度优化（VI 升级）**：
  - 提取了设计师提供的“交叉星轨”源极专属 Logo，并采用 SVG 组件化渲染，保证极致清晰度。
  - 对首屏标题的高亮渐变色进行了深度调优（深空蓝到天际青、赛博紫到深靛蓝的渐变），强化了科技呼吸感。
  - 大幅优化了首屏的排版间距、留白比例以及对齐方式。
  - 将导航栏和底部的文字排版统一调整为“Logo + 源极科技 + OriginX”的横向布局。
- [x] **视界页面（News）重构**：由原来狭窄的单列列表，升级为现代化的高级网格（Grid）卡片布局，文章详情页限制为更适合沉浸式阅读的宽度。
- [x] **生产环境部署**：配置 Nginx 与 PM2，将前后端全量部署到腾讯云服务器。

## 2. 核心功能实现逻辑

### 2.1 源极视界（新闻动态）功能
- **前端展现**：`src/app/pages/News.tsx`。使用了响应式网格布局，通过 axios 调用 `/api/news?status=published` 获取已发布的新闻。
- **后台管理**：`src/app/admin/NewsEditor.tsx` 及 `NewsList.tsx`。提供了富文本 Markdown 编辑器（`@uiw/react-md-editor`），支持插入本地上传的图片。状态分为“草稿(draft)”与“发布(published)”。
- **后端接口**：`server/src/routes/news.js` 提供标准的 RESTful CRUD 接口。

### 2.2 联络中心（客户留言表单）
- **前端交互**：`src/app/pages/Home.tsx` 中底部的 Terminal 区域。使用了 `react-hook-form` 进行表单状态管理和轻量级验证。左侧伪终端展示邮箱与微信联系方式。
- **后端接收**：提交的数据经过 `/api/contacts` 接口，存入 SQLite 的 `contacts` 表。
- **后台处理**：运营人员登录后台（`/admin/contacts`）可查看最新的意向客户需求，并可以标记为“已读”。

### 2.3 全局与页面 SEO 配置
- 采用了 `react-helmet-async`。
- **全局配置**：在后台“全站页面配置”中通过 JSON 调整全局标题、描述和关键字。
- **动态页面**：在查看单篇“源极视界”文章时，系统会自动将网页的 `<title>` 和 `<meta description>` 替换成该文章的标题和摘要，利于搜索引擎收录。

### 2.4 部署架构（Nginx + PM2）
- **代理转发**：所有的 `/api/*` 请求以及上传文件 `/uploads/*` 的请求，都由部署在云服务器的 Nginx （80端口）拦截，并转发给监听在 3000 端口的 Node.js/PM2 进程。
- **静态托管**：打包后的 React 前端代码（dist 目录）由 Nginx 直接高效托管。
