# 源极科技官网 - 新闻与动态简易后台设计方案 (最终确认版)

## 1. 架构选型: 轻量级前后端分离架构

基于国内服务器环境和“轻量、快、省钱”的核心诉求，我们采用**手写极简 Node.js 后端 + SQLite 数据库**的方案。

### 技术栈
*   **前端展示与管理 (Front-end)**：React 18 + Vite (当前已有的项目)
    *   在现有官网项目中新增一个受保护的路由 `/admin`。
    *   运营人员通过该页面登录后，进行文章的增删改查。
*   **后端接口 (Back-end)**：Node.js + Express
    *   一个独立的、极轻量的服务端程序。
    *   提供 RESTful API 供前端调用（如获取文章列表、保存文章、上传图片等）。
*   **数据库 (Database)**：SQLite
    *   **零配置、免安装**。所有数据都存储在后端服务器的一个本地文件（如 `database.sqlite`）中。
    *   **极易备份与迁移**：只需复制这个文件，即可完成全部数据的备份和迁移。
    *   **资源占用极低**：非常适合低配云服务器（1核1G/2G 均可流畅运行）。
*   **文件存储**：本地文件系统 (Local File System)
    *   文章配图和封面直接上传至服务器的指定目录（如 `public/uploads`），并通过静态资源服务对外提供访问。

## 2. 数据库表结构设计 (News 表)

SQLite 数据库核心数据结构设计如下：

| 字段名 (Field) | 数据类型 (Type) | 必填 | 描述 (Description) |
| :--- | :--- | :--- | :--- |
| `id` | INTEGER | 是 | 唯一主键，自增 |
| `title` | TEXT | 是 | 文章标题 |
| `summary` | TEXT | 是 | 文章摘要 (用于新闻列表页展示) |
| `content` | TEXT | 是 | 正文内容 (Markdown 格式) |
| `cover_image` | TEXT | 否 | 封面图 URL 路径 |
| `category` | TEXT | 是 | 分类 (如：`行业洞察`, `产品动态`, `实验室探索`) |
| `status` | TEXT | 是 | 状态：`draft` (草稿), `published` (已发布) |
| `publish_date` | TEXT | 是 | 发布日期 (ISO 8601 格式字符串) |
| `created_at` | TEXT | 是 | 创建时间 (系统自动生成) |
| `updated_at` | TEXT | 是 | 最后修改时间 (系统自动生成) |

## 3. 简易后台管理界面 (Admin UI) 功能规划

在 React 前端项目中新增 `/admin` 路由（需密码登录），包含以下功能页面：

### 3.1 登录页 (`/admin/login`)
- 仅提供给管理员的简单账号密码登录界面（通过 JWT Token 维持会话）。

### 3.2 控制台 / 文章管理列表 (`/admin/news`)
- 表格展示当前所有的文章。
- 展示字段：标题、分类、状态（草稿/已发布）、发布日期。
- 操作按钮：【新增文章】、【编辑】、【删除】。

### 3.3 文章编辑/发布页 (`/admin/news/edit`)
- **基础信息区**：标题、摘要、分类下拉框、发布时间选择器。
- **正文编辑器**：集成轻量级的 Markdown 编辑器（如 `uiw/react-md-editor`），支持所见即所得。
- **图片上传**：支持将图片拖拽或上传到服务器，并自动返回相对路径 URL 插入到正文中。

---

## 4. 后端 API 接口规划

Express 后端将提供以下核心接口 (API)：

*   `POST /api/auth/login`：管理员登录，验证密码并返回 Token。
*   `GET /api/news`：获取新闻列表（支持分页、按状态/分类筛选）。
*   `GET /api/news/:id`：获取单篇新闻详情。
*   `POST /api/news`：创建新文章（需鉴权）。
*   `PUT /api/news/:id`：更新文章内容（需鉴权）。
*   `DELETE /api/news/:id`：删除文章（需鉴权）。
*   `POST /api/upload`：上传图片接口（需鉴权），返回图片的静态访问 URL。

## 5. 开发步骤与计划

我们将分为以下三个阶段进行开发：

**第一阶段：后端搭建与接口开发**
1.  初始化 Node.js + Express 项目结构。
2.  配置 SQLite 数据库并创建数据表。
3.  实现基础的增删改查 API 接口。
4.  实现图片上传和静态资源托管功能。
5.  实现简单的 JWT 登录鉴权接口。

**第二阶段：前端后台界面开发**
1.  在官网项目中安装必要的组件（Markdown 编辑器、Axios 等）。
2.  开发管理员登录页面及路由守卫。
3.  开发后台文章列表页（对接 API 展示数据）。
4.  开发文章编辑页（集成 Markdown 编辑器和图片上传功能）。

**第三阶段：官网展示页动态化对接**
1.  将 `News.tsx` 页面中的硬编码模拟数据，替换为通过 Axios 调用后端 `/api/news` 接口获取的真实数据。
2.  开发动态的新闻详情页路由（如 `/news/:id`），根据 ID 请求并展示完整的 Markdown 文章内容。
