# 源极科技官网 - 项目文件结构目录文档

本项目为一个基于 React (Vite) + Node.js (Express) + SQLite 构建的前后端分离的全栈项目。

## 目录结构（全景概览）

```
科技公司官网设计方案/
├── package.json               # 前端项目依赖与脚本配置
├── vite.config.ts             # Vite 构建配置文件
├── originx.conf               # [线上环境] Nginx 部署配置文件
├── .gitignore                 # Git 忽略规则
├── README.md                  # 项目基础说明
├── src/                       # 📂 前端 React 源代码目录
│   ├── main.tsx               # 前端项目入口文件
│   ├── app/                   # 主业务目录
│   │   ├── App.tsx            # React 根组件 (包裹路由和状态提供者)
│   │   ├── routes.tsx         # 路由表配置文件 (前台与后台路由映射)
│   │   ├── components/        # 公共组件目录
│   │   │   ├── Layout.tsx     # 全局页面框架 (导航栏、移动端菜单、页脚)
│   │   │   ├── SEO.tsx        # 动态 SEO 元数据管理组件
│   │   │   ├── Logo.tsx       # 核心：源极交叉星轨 SVG 组件
│   │   │   └── ui/            # UI 组件库库文件 (shadcn/ui, Radix)
│   │   ├── pages/             # 🌍 前台公开展示页面
│   │   │   ├── Home.tsx       # 首页 (大屏展示、三大矩阵、联络中心表单)
│   │   │   ├── Brand.tsx      # 关于源极 (品牌理念页)
│   │   │   └── News.tsx       # 源极视界 (新闻列表页、新闻阅读页)

├── server/                    # 📂 后端 Node.js + Express 目录
│   ├── package.json           # 后端项目依赖 (express, sqlite3, multer, jsonwebtoken)
│   ├── src/

│   │   ├── db.js              # SQLite 数据库初始化与连接池 (建表逻辑)
```

### 关键目录说明
1. **`src/app/pages/` 与 `src/app/admin/`**：代表了前后台彻底的逻辑分离，公用同一个 React 单页应用内核。
2. **`server/`**：一套独立的小型后端服务，使用 `npm run dev` 或 `node src/index.js` 即可启动，与前端的 Vite 服务器互不干扰。
3. **`server/data/database.sqlite`**：这是整个网站的灵魂文件。如果需要备份网站数据（文章、设置、客户信息），只需定期下载并备份此单一文件即可。
