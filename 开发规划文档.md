# 源极科技 (OriginX) 官网 - 下一步开发规划

## 1. 动态数据化与简易后台接入 (CMS)
**目标**：将新闻和洞察内容由静态数据改为动态数据，让运营人员可以随时发布和调整文章。
**方案**：方案 B - 接入 Headless CMS (如 Supabase / Strapi) 或自建简单的后台 API。
**步骤**：
- 在开始正式开发前，将基于 `@后台设计.md` 出具一份**简易后台架构和数据库设计方案**，与需求方对齐后再进行开发。
- 开发简易后台，支持文章的增删改查 (富文本或 Markdown 格式)。
- 改造 `News.tsx`，对接后台 API 获取动态数据。

## 2. 完善“联络中心”交互
**目标**：提升首页底部的联络中心转化率，目前仅有文本展示。
**方案**：增加一个实际的留言/需求提交表单。
**步骤**：
- 利用已安装的 `react-hook-form` 开发表单组件 (包含姓名、公司、联系方式、需求描述等)。
- 结合后端的邮件推送服务 (如 Resend, EmailJS) 或自建 API，让客户提交的信息直接推送到指定的业务邮箱。

## 3. 移动端适配与细节打磨
**目标**：确保在手机等移动设备上拥有完美的体验。
**步骤**：
- **移动端导航**：修改 `Layout.tsx`，当处于手机端屏幕尺寸时，将顶部导航栏替换为汉堡菜单 (Hamburger Menu)，使用 shadcn/ui 的 `Sheet` 弹出抽屉组件实现。
- **动效审查**：检查并优化各个板块的动效在移动端的性能表现，确保排版不越界、动画不卡顿。

## 4. SEO 与网站元数据 (高优)
**目标**：提升搜索引擎的抓取排名，以及社交媒体分享时的卡片展现效果。
**方案**：引入 `react-helmet-async`。
**步骤**：
- 安装并配置 `react-helmet-async`。
- 为首页、品牌页以及每一篇单独的新闻文章动态注入不同的 `<title>`、`<meta name="description">`。
- 配置 OpenGraph (OG) 标签 (og:title, og:description, og:image)，利于微信、朋友圈和百度/谷歌抓取。
